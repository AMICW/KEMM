## 2. 参考模板对照后的主要缺项

### 2.1 参考技术交底书模板后发现的缺项

从技术交底书模板看，除了“技术问题、技术方案、技术效果”外，通常还会补得很细：

- 一段较正式的摘要
- 明确的步骤化技术方案
- 每个步骤的进一步限定
- 相关公式的解释
- 技术效果与现有技术对比
- 具体实施方式的文字展开
- 相关附图列表

当前我们的交底书已经覆盖其中大部分，但还可以补强：

- 一版正式摘要，不只是“简版摘要”
- 一组与附图一一对应的“图1-图N”草案说明
- 更明确的“本发明相对于最接近现有技术的区别特征”
- 一段“本发明可推广到无人船、岸基辅助决策、协同避碰”的应用延展说明

### 2.2 参考专利初稿模板后发现的缺项

从专利初稿模板看，真正用于递交前整理的说明书通常会再补：

- 更具体的物理模型参数
- 明确的实验或仿真设置
- 对比例或基线方法
- 结果分析段落
- 附图和图注逐一列明
- 每个实施例与图的对应关系

当前我们的说明书已经有公式和机制，但还欠缺更强的“实施例落地感”，主要表现在：

- 没有完整参数表
- 没有对比例实施方式
- 没有一组可直接写入说明书的结果表
- 没有明确指定每幅图该画什么

## 3. 说明书层面还需要补什么

### 3.1 必补项

以下内容建议补入说明书正式版或配套材料中：

1. 最接近现有技术的引证文献

当前 [patent_specification_ship_kemm_draft.md](d:/VS%20Code/code/KEMM%28renew%29/docs/patent_specification_ship_kemm_draft.md#L29) 已写了背景技术问题，但还没有把最接近的现有技术文献名称、作者、专利号或论文信息补进去。

建议至少补三类引证：

- 一类静态/准静态多目标船舶轨迹规划方法
- 一类动态重启式或旧种群复用式动态规划方法
- 一类相关的知识迁移或预测增强动态优化方法

这样做的作用是：

- 让背景技术更符合 CNIPA 对“检索、理解、审查有用”的要求
- 让区别特征更明确
- 后续代理人更容易组织创造性答复

2. 正式摘要文本

虽然说明书草稿中已有 [可直接转摘要与发明内容的简版文本](d:/VS%20Code/code/KEMM%28renew%29/docs/patent_specification_ship_kemm_draft.md#L1517)，但还不是最终摘要格式。

正式摘要还应满足：

- 写明名称和所属技术领域
- 明确所解决的技术问题
- 以技术方案为主
- 写明主要用途
- 文字部分不超过 300 字
- 不使用商业性宣传用语

3. 摘要附图指定方案

当前材料还没有指定“哪一幅图作为摘要附图”。

建议优先选：

- 总体流程图

原因是：

- 最能体现技术方案主链
- 比复杂公式图更适合作为摘要附图
- 更符合方法专利的常见做法

4. 附图编号与图面方案

当前 [patent_specification_ship_kemm_draft.md](d:/VS%20Code/code/KEMM%28renew%29/docs/patent_specification_ship_kemm_draft.md#L217) 只有附图说明思路，但还没有形成真正可画图的附图方案。

建议补成至少以下 8 幅：

- 图1：本发明整体方法流程图
- 图2：系统总体架构图
- 图3：环境变化诊断与自适应分配流程图
- 图4：记忆、预测、迁移、重初始化协同生成候选图
- 图5：统一候选池与约束支配环境选择图
- 图6：滚动重规划执行图
- 图7：交叉会遇场景示意图
- 图8：港口高密障碍场景示意图

5. 附图标记体系

当前文本没有形成统一的图面标号系统。

建议在最终图纸中统一使用如下附图标记：

- `100` 数据获取模块
- `110` 轨迹建模模块
- `120` 变化诊断模块
- `130` 自适应分配模块
- `140` 历史记忆模块
- `150` 前沿预测模块
- `160` 迁移模块
- `170` 重初始化模块
- `180` 候选池构建模块
- `190` 环境选择模块
- `200` 轨迹输出执行模块
- `210` 本船
- `220` 目标船
- `230` 静态障碍物
- `240` 环境场

这样后续说明书、附图和装置权利要求更容易统一。

### 3.2 强烈建议补的项

1. 参数范围和推荐值

当前说明书公式很多，但正式审查和后续实施充分性更需要“参数范围 + 推荐值”。

建议至少补以下参数：

- 规划时域 `H_plan` 的范围和推荐值
- 执行时域 `H_exec` 的范围和推荐值
- 种群规模 `N`
- 各候选生成机制的最小配额或最小比例
- 记忆库容量范围
- 相似环境检索数量 `top-K`
- 预测窗口长度
- 子空间维度选择阈值
- 安全净距阈值
- 硬侵入阈值定义

2. 一组完整的优选实施例参数表

模板专利初稿的特点是，真正进入“具体实施方式”时，会出现很多明确数值。

当前我们的说明书虽然讲明了机制，但少一张“实施例参数表”。

建议至少做两张表：

- 表1：算法参数表
- 表2：场景参数表

表1 可包含：

- 种群规模
- 进化代数
- 记忆候选上限
- 预测候选上限
- 迁移候选上限
- 重初始化候选上限
- 变化阈值
- 预测置信度阈值
- 相似度阈值

表2 可包含：

- 本船初始位置、航向、航速
- 目标船初始位置、航向、航速
- 障碍物数量
- 障碍物尺寸
- 风流场取值
- 目标位置

3. 对比例实施方式

专利里不是必须写实验对比，但如果你已经有结果，写“对比例实施方式”很有价值。

建议至少加入：

- 对比例1：完全重初始化
- 对比例2：仅保留旧种群
- 对比例3：不使用前沿预测

这样在技术效果部分可以更自然写出：

- 恢复更快
- 风险更低
- 搜索更稳定

4. 结果表述

当前 [patent_specification_ship_kemm_draft.md](d:/VS%20Code/code/KEMM%28renew%29/docs/patent_specification_ship_kemm_draft.md#L1358) 和 [patent_specification_ship_kemm_draft.md](d:/VS%20Code/code/KEMM%28renew%29/docs/patent_specification_ship_kemm_draft.md#L1392) 已有实施例，但结果仍偏概念化。

建议补成更像专利初稿模板的写法：

- 给出每个场景的参数
- 给出风险、时间、燃油等结果
- 给出对比例结果
- 给出“从结果可见……”式的技术效果分析

5. 部署实施方式

当前说明书主要是算法和仿真层。

建议再补 1 小节，说明该方法可部署为：

- 船载终端部署
- 岸基辅助决策部署
- 云端规划服务部署
- 船岸协同部署

写清楚：

- 哪些模块本地运行
- 哪些模块可远程运行
- 数据如何传输
- 控制结果如何下发

这会让装置和电子设备权利要求更扎实。

6. 异常或保底执行逻辑

从工程角度讲，船舶规划方法最好写明：

- 当当前周期无满足安全要求的候选轨迹时的保底策略
- 当传感数据缺失或环境指纹不足时的回退策略
- 当预测置信度低时的回退策略

这一点在说明书中已有部分涉及，但还可以写得更完整。

建议明确：

- 低置信预测回退到线性外推或重初始化
- 候选解全部不可行时触发保守安全轨迹
- 仅执行局部前段，下一周期重新评估

## 7. 申请提交层面还需要补什么

这是模板之外，但正式申请一定会碰到的部分。

### 7.1 申请文件包

正式递交通常还需要整理以下文件或信息：

- 请求书
- 说明书
- 权利要求书
- 摘要
- 摘要附图
- 说明书附图
- 申请人信息
- 发明人信息
- 联系人信息
- 专利代理委托材料

### 7.2 需要提前确认的行政事项

建议你和老师尽早确认：

- 发明人排序
- 是否已有论文、组会汇报、公开答辩材料可能影响新颖性
- 是否需要先做保密审查

### 7.3 命名与统一性

正式申请前建议统一：

- 发明名称
- 模块名称
- 风险项名称
- 约束项名称
- 本船/目标船/障碍物/环境场的叫法

不要在不同文档中混用：

- “risk / 风险目标 / collision_risk / 综合风险”
- “ship / 船舶 / 本船侧”
- “candidate pool / 统一候选池 / 多源候选池”

### 8.1 说明书中还存在内部写作痕迹

例如：

- 仓库文件链接

- “优选地”

- “可选地”
- “其中”

- “主线”
- “benchmark-only”
- “debug”
- “fallback 先这样”

根据 CNIPA 要求：

- 图号应按 `图1，图2，……` 顺序编号

## 12. 官方要求摘录结论

结合 CNIPA 公开要求，当前需要重点记住：

1. 说明书应按顺序包含：

- 发明名称
- 技术领域
- 背景技术
- 发明内容
- 附图说明
- 具体实施方式

2. 摘要应当：

- 写明名称和所属技术领域
- 清楚反映技术问题、技术方案要点和主要用途
- 不超过 300 字
- 指定摘要附图

3. 权利要求书应当：

- 记载技术特征
- 用语与说明书一致
- 清楚、简要
- 从属权利要求只能引用在前的权利要求
