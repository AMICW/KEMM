## 一、技术领域

本发明涉及智能航运、船舶自动避碰、路径规划、计算机辅助控制以及动态多目标优化技术领域，尤其涉及一种面向动态航行环境的船舶多目标轨迹优化方法、装置、电子设备及计算机程序产品。

更具体地，本发明适用于如下技术场景：

- 本船在存在目标船、静态障碍物、航道边界、海流、风场和环境风险场的动态环境中航行；
- 规划系统需要在滚动重规划框架下重复输出可执行轨迹或控制指令；
- 规划系统需要同时优化燃油消耗、航行时间和碰撞风险，并处理安全距离、越界和终端偏差等约束；
- 当环境变化时，系统需要在较短时间内恢复有效搜索，而不能仅依赖完全随机重启或机械保留旧解。

## 二、背景技术

### 2.1 现有船舶轨迹规划技术概述

当前船舶轨迹规划和自动避碰领域，常见的技术路线大致可以分为以下几类。

#### 2.1.1 静态或准静态多目标优化方法

该类方法通常在单个规划周期内假设环境相对稳定，通过遗传算法、粒子群算法、非支配排序进化算法或启发式搜索，对燃油、时间和安全性进行联合优化。

这类方法的优点是：
- 数学形式清晰；
- 在静态环境中容易得到较好的 Pareto 解集；
- 便于做多目标取舍。

但其缺点是：
- 对环境变化不敏感；
- 当目标船状态、海流、障碍物影响范围或风险场发生变化时，原有最优解可能迅速失效；
- 若重新完整求解，恢复时间长，不适合滚动重规划。

#### 2.1.2 完全重启式动态规划方法

这类方法在检测到环境变化后，直接重新初始化种群或重新规划整条航迹，以避免旧解对当前环境造成误导。

该类方法的优点是：
- 不容易被旧环境的无效解结构拖累；
- 实现简单。

其缺点在于：
- 每次变化都从零开始，导致恢复慢；
- 大量计算资源浪费在重复搜索上；
- 在船舶滚动重规划场景下，可能无法满足实时或准实时要求。

#### 2.1.3 旧种群直接复用或轨迹延续方法

这类方法在环境变化后直接保留上一时刻种群、局部轨迹或者控制参数，在其基础上继续搜索。

该类方法的优点是：
- 初始搜索位置更接近上一时刻的有效解；
- 在环境变化较小时收敛较快。

其缺点在于：
- 当变化较大时，旧种群会把搜索拖向错误方向；
- 容易产生风险较高或不可执行的候选解；
- 缺乏对“旧知识是否仍有价值”的判断机制。

### 2.2 现有技术存在的核心问题

本发明认为，现有技术至少存在以下五类技术问题。

#### 2.2.1 缺乏变化诊断能力

现有方案通常将所有环境变化一概而论，没有区分小幅漂移、连续变化、中等尺度结构变化和剧烈突变。

但从工程和优化角度看，不同变化模式所对应的最佳恢复策略并不相同：
- 小幅连续漂移更适合利用历史信息和趋势预测；
- 中等变化更适合跨环境迁移；
- 剧烈变化更需要重新探索。

如果不区分变化类型，恢复策略就容易失配。

#### 2.2.2 缺乏高效历史知识复用方式

直接保存旧种群存在两类问题：
- 存储冗余高，尤其在长期滚动重规划中会快速膨胀；
- 旧种群中包含大量低质量样本，直接复用会把噪声带回当前搜索。

而直接保留少量原始精英解又会导致表达能力不足，不能充分刻画历史前沿结构。

因此，现有技术需要一种既能压缩存储、又能在需要时恢复结构化候选解的记忆机制。

#### 2.2.3 缺乏对前沿演化趋势的建模

很多动态环境不是完全随机变化，而是沿时间连续演化。例如：
- 目标船逐步靠近或远离；
- 海流逐步增强或减弱；
- 风险区域随环境场发生漂移。

如果只基于当前时刻做被动搜索，而不利用历史前沿的时间演化规律，则会浪费可预测性。

因此，现有技术需要一种能够对 Pareto 前沿整体演化趋势进行建模的机制。

#### 2.2.4 缺乏统一调度多种恢复机制的能力

即使部分方案引入了预测、迁移或历史复用，常见做法也往往是固定比例、经验性切换或硬编码规则。

这种做法存在以下问题：
- 需要人工调节；
- 场景切换时失稳；
- 难以解释某一策略在当前环境中为何有效或为何失败；
- 难以在动态环境中长期自适应优化。

因此，需要一种能够根据环境变化强度和历史效果自动调整各恢复机制预算的技术方案。

#### 2.2.5 在船舶场景中安全与效率难兼顾

船舶轨迹规划属于典型的技术问题，其输入和输出都具有明确物理意义：
- 输入包括本船状态、目标船状态、障碍物和环境场；
- 输出包括航路点、速度参数和控制结果。

该类问题不仅要追求航行效率和燃油消耗，还要确保动态安全净距和避碰效果。

如果环境变化后恢复策略不合理，会直接导致：
- 最小净距不足；
- 碰撞风险上升；
- 轨迹越界；
- 终端偏差增大；
- 控制不平滑。

因此，需要一种真正适用于动态航行环境、能在安全与效率之间取得平衡的轨迹优化方法。

### 2.3 本发明要解决的技术问题

针对上述缺陷，本发明要解决的技术问题是：

在动态航行环境下，当本船周围目标船状态、静态障碍物影响范围、航道条件或者环境场发生变化时，如何在滚动重规划过程中快速恢复轨迹搜索，并兼顾燃油消耗、航行时间和碰撞风险，避免现有技术中完全重启导致恢复慢以及机械复用旧解导致安全性下降的问题。

## 三、发明内容

### 3.1 总体技术构思

本发明提出一种面向动态航行环境的船舶多目标轨迹优化技术方案。

该方案的核心思想不是在环境变化后简单重新搜索，而是：

1. 先对当前环境变化进行定量诊断；
2. 再根据变化信息自适应地分配多类候选解恢复机制；
3. 通过历史记忆、前沿预测、跨环境迁移和重初始化共同生成候选解；
4. 将各来源候选解统一放入候选池；
5. 通过带约束支配的环境选择保留当前规划解集；
6. 选取代表轨迹并执行滚动重规划。

换言之，本发明不是单一预测算法、单一记忆算法或单一进化算法，而是一种在动态技术环境下统一调度多种知识来源的船舶轨迹恢复与优化框架。

### 3.2 技术方案概述

本发明的技术方案可概括为以下步骤：

S100，获取本船、目标船、静态障碍物及环境场数据；

S200，根据所述数据构建动态多目标轨迹优化模型，所述模型至少包括燃油目标、时间目标和风险目标，并建立约束违背度指标；在船舶轨迹规划实施方式中，还可根据起点、目标点、静态障碍物和目标船空间关系构造场景感知的绕行初始候选生成依据；

S300，在环境变化前或变化响应开始时归档当前环境的精英解集、前沿特征和种群结构信息；

S400，根据当前时刻与历史时刻的前沿特征确定环境变化信息，所述环境变化信息包括变化幅度、知识可迁移性和预测置信度中的至少一种；

S500，根据所述环境变化信息，自适应确定历史记忆候选、前沿预测候选、跨环境迁移候选和重初始化候选之间的分配比例；并在优选实施方式中确定场景感知绕行初始候选的注入数量、偏移尺度或保留比例；

S600，基于历史记忆候选生成机制生成记忆候选；

S700，基于前沿预测候选生成机制生成预测候选；

S800，基于跨环境迁移候选生成机制生成迁移候选；

S900，结合重初始化候选、场景感知绕行初始候选以及必要的精英保留候选和历史种群保留候选，构建统一候选池；

S1000，对所述统一候选池执行约束支配环境选择，得到当前规划解集；

S1100，从所述规划解集中按照障碍侵入情况、安全净距满足情况、终端推进程度和多目标折中得分选取代表轨迹，输出轨迹参数或控制结果，并在滚动重规划框架下执行局部前段轨迹。

### 3.3 有益效果

与现有技术相比，本发明至少具有以下技术效果。

#### 3.3.1 环境变化后恢复更快

本发明通过历史记忆、前沿预测和跨环境迁移的协同作用，使候选解在变化发生后能够更快靠近新的有效前沿，减少完全随机重启带来的搜索延迟。

#### 3.3.2 安全性更强

本发明通过风险目标和约束违背度的统一支配比较，避免将安全问题简单并入普通目标函数而被掩盖，从而更利于保留满足安全净距和边界约束的轨迹。

#### 3.3.3 对不同变化模式适应性更强

通过变化诊断与自适应分配机制，本发明能够针对小变化、连续漂移、中等结构变化和剧烈变化采用不同恢复策略，提高鲁棒性。

#### 3.3.4 计算开销更合理

通过记忆压缩、相似环境检索和多源迁移，本发明减少了重复搜索和无效重启，能够在滚动重规划中提升计算资源利用率。

#### 3.3.5 结构清晰、便于扩展

本发明将记忆、预测、迁移和重初始化四类机制统一纳入候选池框架，支持模块替换和平台扩展，便于工程部署。

## 五、具体实施方式

以下结合可选实施例、系统模块和底层公式对本发明作进一步详细说明。应当理解，这些实施方式仅用于解释本发明，而不用于限定本发明的保护范围。

### 5.1 术语定义与建模约定

#### 5.1.1 时间索引

设滚动重规划过程离散为若干规划时刻，记为：

`t = 0, 1, 2, ..., T`

其中：
- `t` 可以表示实际物理时间；
- 也可以表示第 `t` 个重规划步；
- 在动态优化语境下，`t` 同时表示环境状态的演化索引。

#### 5.1.2 决策变量

在本发明的优选实现中，轨迹由若干中间航路点和各航段速度共同表示。

设存在 `K` 个中间航路点，则一个决策向量可表示为：

`x = [w_1^x, w_1^y, v_1, w_2^x, w_2^y, v_2, ..., w_K^x, w_K^y, v_K]`

其中：
- `w_i^x, w_i^y` 表示第 `i` 个中间航路点坐标；
- `v_i` 表示第 `i` 段航速参数。

对上述决策变量设定边界：

`l <= x <= u`

其中：
- `l` 为下界；
- `u` 为上界。

#### 5.1.3 多目标优化问题

本发明求解的动态多目标优化问题可写为：

`min F(x,t) = [f_fuel(x,t), f_time(x,t), f_risk(x,t)]`

同时伴随约束违背度函数：

`cv(x,t) >= 0`

其中：
- `f_fuel(x,t)` 表示燃油目标；
- `f_time(x,t)` 表示时间目标；
- `f_risk(x,t)` 表示碰撞风险目标；
- `cv(x,t)` 表示安全和边界等约束的综合违背度。

#### 5.1.4 风险分解

本发明优选地将风险分解为以下部分：
- 船舶域风险；
- DCPA/TCPA 风险；
- 静态障碍物风险；
- 环境风险；
- 安全净距不足与硬侵入惩罚；
- 风险侵入时间惩罚。

其中，终端推进不足在优选实施方式中不直接并入风险维，而是优选通过燃油目标、时间目标中的终端距离惩罚项以及代表轨迹选择阶段的终点推进排序体现。该分解方式有利于增强解释性，避免将“尚未充分接近目标点”与“真实碰撞风险暴露”混为同一种风险信号。

### 5.2 系统总体架构

本发明优选的系统架构包括如下模块：

1. 数据获取模块
- 获取本船状态、目标船状态、障碍物信息和环境场信息。

2. 问题建模模块
- 将航行环境转化为动态多目标轨迹优化问题。

3. 环境变化诊断模块
- 通过前沿特征和历史信息估计变化强度、可迁移性和预测置信度。

4. 自适应分配模块
- 根据变化信息确定各恢复机制的预算比例。

5. 历史记忆模块
- 对历史精英结构进行压缩存储，并按环境相似度检索恢复。

6. 前沿预测模块
- 建模历史 Pareto 前沿特征序列并预测未来前沿趋势。

7. 迁移模块
- 将相似历史环境中的精英解通过结构映射迁移到当前环境。

8. 重初始化模块
- 在知识不足或变化剧烈时补充探索能力。

9. 候选池构建模块
- 将多类候选解合并为统一候选池。

10. 环境选择模块
- 执行约束支配环境选择，保留当前规划解集。

11. 代表轨迹输出模块
- 从当前规划解集中选取代表轨迹并输出控制结果。

12. 滚动重规划执行模块
- 执行局部轨迹前段，更新环境状态并触发下一次规划。

在优选的船舶场景实施方式中，所述系统架构还可进一步包括或隐含包含场景感知初始候选子单元；所述子单元可以作为候选池构建模块的前置子单元部署，也可以作为求解器初始化阶段的启发式注入单元部署，其作用是在统一环境选择之前，先提供若干具有明确绕行语义的 warm-start 候选。

#### 5.2.1 场景感知初始候选子单元

在一种优选实施方式中，设本船起点和目标点分别为 `p_s` 与 `p_g`，则航迹主方向可写为：

`e = (p_g - p_s) / ||p_g - p_s||`

与其正交的横向单位向量可写为：

`n = [-e_y, e_x]`

设局部规划中存在 `K` 个中间航路点，其归一化进度为 `rho_i in (0,1)`，则直航骨架点可写为：

`p_i^base = p_s + rho_i * (p_g - p_s)`

为生成具有连续绕行趋势的候选，可对所述直航骨架点沿横向单位向量施加包络偏移，得到走廊式绕行候选：

`p_i^corr = p_i^base + sigma * A * sin(pi * rho_i) * n`

其中：
- `sigma` 取 `+1` 或 `-1`，表示左右两侧绕行方向；
- `A` 为与安全净距阈值相关的偏移尺度。

进一步地，设某一静态障碍物或动态目标船对应的危险中心为 `p_h`，其在主航向上的归一化投影进度为 `rho_h`，则可构造针对局部危险区抬升的高斯型偏移包络：

`w_i^haz = exp(-((rho_i - rho_h)^2) / eta)`

并得到危险区引导绕行候选：

`p_i^haz = p_i^base + sigma * A_h * w_i^haz * n`

其中：
- `eta` 为局部包络宽度参数；
- `A_h` 为与危险体尺度和安全净距相关的局部绕行偏移量。

优选地，所述场景感知初始候选子单元至少利用以下信息中的两项生成候选：

- 起点与目标点之间的主航向几何关系；
- 静态障碍物的中心位置、包络半径或边界外接尺度；
- 目标船的初始位置或预测局部占用区域；
- 安全净距阈值；
- 航路点数量与速度上下界。

所述场景感知初始候选子单元的技术作用在于：

- 在优化初期显式提供“安全绕行而非直穿危险区”的可解释候选；
- 减少交叉会遇和港口高密障碍场景下高风险直航候选对种群的主导；
- 使统一候选池在进入约束支配环境选择之前，就具备更强的安全先验和几何可行性。

上述优选实现可对应 [problem.py](d:/VS%20Code/code/KEMM%28renew%29/ship_simulation/optimizer/problem.py#L181) 与 [kemm_solver.py](d:/VS%20Code/code/KEMM%28renew%29/ship_simulation/optimizer/kemm_solver.py#L146)。

### 5.3 船舶运动模型

本发明优选使用带速度滞后和航路点跟踪能力的增强型 Nomoto 一阶船模。

设本船状态为：

`s = (x, y, psi, v, r)`

其中：
- `x, y` 为平面坐标；
- `psi` 为航向角；
- `v` 为航速；
- `r` 为偏航角速度。

对于目标航路点，先计算期望航向：

`psi_d = atan2(y_target - y, x_target - x)`

航向误差为：

`e_psi = wrap_to_pi(psi_d - psi)`

命令偏航角速度：

`r_cmd = clip(k_h * e_psi, -r_cmd_max, r_cmd_max)`

其中：
- `k_h` 为航向控制增益；
- `r_cmd_max` 为最大命令偏航角速度。

Nomoto 一阶动态可写为：

`dot(r) = (K_n * r_cmd - r) / T_n`

其中：
- `K_n` 为 Nomoto 增益；
- `T_n` 为 Nomoto 时间常数。

航速一阶滞后模型：

`dot(v) = (v_target - v) / T_v`

其中 `T_v` 为速度时间常数。

考虑环境漂移速度 `d(t)` 后，本船地面速度为：

`v_g = [v cos(psi), v sin(psi)] + d(t)`

于是位置更新为：

`x_{k+1} = x_k + v_g^x * dt`

`y_{k+1} = y_k + v_g^y * dt`

上述优选实现对应 [ship_model.py](d:/VS%20Code/code/KEMM%28renew%29/ship_simulation/core/ship_model.py#L62)。

该模型的技术意义在于：
- 不将本船简化为瞬时可达点；
- 保留速度滞后和转向动力学；
- 能更真实地反映控制输出与轨迹可执行性的关系。

### 5.4 燃油目标建模

本发明优选采用可比较的燃油代理模型，而非依赖具体船型的机舱工况表。

设瞬时燃油率为：

`q_f(v, r) = c_b * v^3 * (1 + c_e * E_env) * (1 + c_t * |r|)`

其中：
- `c_b` 为基础燃油系数；
- `c_e` 为环境惩罚系数；
- `E_env` 为环境负荷项；
- `c_t` 为操纵附加损耗系数；
- `|r|` 反映转向剧烈程度。

环境负荷项优选可写为：

`E_env = v_current + k_w * v_wind`

其中：
- `v_current` 表示海流速度标量；
- `v_wind` 表示风速；
- `k_w` 为风漂系数。

总燃油目标通过轨迹积分得到：

`f_fuel = integral q_f(v(t), r(t)) dt`

在离散实现中可写为：

`f_fuel ≈ sum q_f(v_k, r_k) * Delta t_k`

在滚动重规划实施方式中，为避免轨迹在局部规划时域末端虽然节能但未能继续向目标点推进，优选再引入燃油维终端距离惩罚：

`P_f_terminal = lambda_ft * d_terminal`

其中：
- `lambda_ft` 为燃油维终端惩罚系数；
- `d_terminal` 为轨迹终点到目标点的距离。

若进一步考虑安全净距不足对能耗决策的间接影响，则燃油目标可优选表示为：

`f_fuel = f_fuel_base + P_f_terminal + lambda_fs * P_safe`

其中 `f_fuel_base` 为基础燃油积分项，`lambda_fs` 为燃油维安全惩罚权重。

该模型对应 [fuel_model.py](d:/VS%20Code/code/KEMM%28renew%29/ship_simulation/core/fuel_model.py#L21)。

该燃油模型的技术作用不在于拟合绝对油耗值，而在于：
- 为优化提供连续、可比较且与速度、操纵、环境负荷相关的技术指标；
- 使能耗与轨迹执行特性建立明确联系。

### 5.5 时间目标建模

时间目标优选采用轨迹执行总时间、终端距离惩罚与安全惩罚的组合。

基础时间目标为：

`f_time_base = t_end - t_start`

为抑制轨迹在局部规划期末未能有效接近目标点的情况，引入终端距离惩罚：

`P_time_terminal = lambda_tt * d_terminal`

其中：
- `lambda_tt` 为时间维终端惩罚系数；
- `d_terminal` 为轨迹终点到目标点的距离。

同时，若最小安全净距低于安全阈值，则引入安全惩罚：

`P_safe = lambda_sc * max(0, C_safe - C_min) + lambda_hi * max(0, -C_min)`

其中：
- `C_safe` 为安全净距阈值；
- `C_min` 为轨迹最小净距；
- `lambda_sc` 为软净距不足惩罚系数；
- `lambda_hi` 为硬侵入惩罚系数。

时间目标可优选表示为：

`f_time = f_time_base + P_time_terminal + lambda_ts * P_safe`

其中 `lambda_ts` 为时间维安全惩罚权重。

由此，燃油目标和时间目标共同承担“局部时域内仍需持续向目标点推进”的语义，而不需要再把终端推进不足并入风险维。

### 5.6 风险目标建模

本发明优选使用分解式风险模型，以增强解释性和安全性。

#### 5.6.1 船舶域风险

设目标船相对本船的位置向量为：

`p_rel = p_target - p_own`

将其旋转到本船体坐标系下：

`p_body = R(psi) * p_rel`

其中 `R(psi)` 为按本船航向角构造的旋转矩阵。

根据相对方位确定前后向和左右向船舶域尺度：

`a_x = gamma_lon * L_ship`

`a_y = gamma_lat * B_ship`

其中：
- `L_ship` 为船长；
- `B_ship` 为船宽；
- `gamma_lon, gamma_lat` 为方向相关放缩系数。

归一化距离定义为：

`d_norm = sqrt((p_body^x / (a_x * m_s))^2 + (p_body^y / (a_y * m_s))^2)`

其中 `m_s` 为软安全边界放大系数。

瞬时船舶域风险定义为：

当 `d_norm <= 1` 时：

`r_domain = 1 + (1 - d_norm)`

当 `d_norm > 1` 时：

`r_domain = exp(-k_d * (d_norm - 1))`

其中 `k_d` 为衰减参数。

该模型对应 [collision_risk.py](d:/VS%20Code/code/KEMM%28renew%29/ship_simulation/core/collision_risk.py#L42)。

#### 5.6.2 DCPA/TCPA 风险

设相对位置和相对速度分别为：

`p_r = p_target - p_own`

`v_r = v_target - v_own`

则最近会遇时间 TCPA 为：

`TCPA = max(0, - (p_r · v_r) / ||v_r||^2 )`

最近会遇距离 DCPA 为：

`DCPA = || p_r + TCPA * v_r ||`

基于空间和时间衰减，DCPA/TCPA 风险定义为：

`r_dcpa = exp(-DCPA / C_safe) * exp(-TCPA / tau_tcpa)`

其中：
- `C_safe` 为安全净距；
- `tau_tcpa` 为 TCPA 衰减时间常数。

该模型对应 [collision_risk.py](d:/VS%20Code/code/KEMM%28renew%29/ship_simulation/core/collision_risk.py#L153)。

#### 5.6.3 静态障碍物风险

设本船当前位置到最近障碍物边界的净距为 `C_obs`。

若 `C_obs <= 0`，说明本船进入障碍区或边界区，则定义：

`r_obs = 2`

若 `C_obs > 0`，则定义：

`r_obs = exp(-C_obs / C_safe)`

该形式实现了如下效果：
- 越接近障碍物，风险越高；
- 进入障碍区时风险急剧上升。

#### 5.6.4 环境风险

设环境风险场标量值为 `r_env_field(p,t)`。

在位置 `p` 和时刻 `t` 处的环境风险可写为：

`r_env = min(r_env_max, r_env_field(p,t))`

其中 `r_env_max` 为环境风险上限，用于防止单项风险无界放大。

#### 5.6.5 风险聚合

对单个时刻，综合风险优选可表示为：

`r_comb = (w_d * r_domain + w_c * r_dcpa + w_o * r_obs + w_e * r_env) * s_colreg`

其中：
- `w_d` 为船舶域风险权重；
- `w_c` 为 DCPA/TCPA 风险权重；
- `w_o` 为障碍物风险权重；
- `w_e` 为环境风险权重；
- `s_colreg` 为基于会遇类型的 COLREG 风险尺度因子。

不同会遇角色可使用不同 `s_colreg`，例如：
- 对遇；
- 交叉让路；
- 交叉直航；
- 追越。

#### 5.6.6 风险目标函数

设轨迹风险序列的最大值和均值分别为：

`r_max = max_k r_comb(k)`

`r_mean = mean_k r_comb(k)`

设风险侵入时间为：

`T_intr = sum I(r_comb(k) >= 1) * Delta t`

则本发明优选的风险目标函数可写为：

`f_risk = alpha * r_max + (1 - alpha) * r_mean + lambda_rs * P_safe + lambda_intr * T_intr`

其中：
- `alpha` 为最大风险和平均风险的平衡系数；
- `lambda_rs` 为风险维安全惩罚权重；
- `lambda_intr` 为侵入时间惩罚系数。

在优选实施方式中，`f_risk` 用于表达沿轨迹的真实风险暴露强度，而不直接承担终端推进不足的语义；终端距离 `d_terminal` 可由燃油目标、时间目标或代表轨迹选择阶段单独体现。

该风险聚合与目标组合对应 [problem.py](d:/VS%20Code/code/KEMM%28renew%29/ship_simulation/optimizer/problem.py#L325)。

### 5.7 约束违背度建模

为在环境选择过程中优先保证安全和可执行性，本发明优选构造约束违背度：

`cv = P_bound + P_safe + P_intr`

其中：
- `P_bound` 为变量越界惩罚；
- `P_safe` 为净距不足和硬侵入惩罚；
- `P_intr` 为侵入时间惩罚。

更具体地：

`P_bound = lambda_b * || x - clip(x,l,u) ||_1`

`P_safe = lambda_sc * max(0, C_safe - C_min) + lambda_hi * max(0, -C_min)`

`P_intr = lambda_intr * T_intr`

其中：
- `lambda_b` 为越界惩罚系数；
- `lambda_sc` 与 `lambda_hi` 分别对应软净距不足惩罚系数和硬侵入惩罚系数；
- `lambda_intr` 为侵入时间惩罚系数。

该形式对应 [problem.py](d:/VS%20Code/code/KEMM%28renew%29/ship_simulation/optimizer/problem.py#L286)。

该设计的技术意义在于：
- 将安全和边界等约束因素显式抽离出来；
- 避免将终端推进不足与安全可行性混写为同一种约束违背；
- 使环境选择时能优先比较可行性，再比较目标最优性；
- 避免简单线性加权导致 Pareto 语义失真。

### 5.8 基础多目标进化骨架

本发明优选在 NSGA-II 式动态多目标进化骨架上构建变化响应机制。

其基础流程包括：

1. 初始化种群；
2. 评价种群；
3. 交叉和变异产生后代；
4. 合并父代和子代；
5. 执行非支配排序；
6. 基于拥挤距离保留多样性；
7. 完成环境选择。

#### 5.8.1 约束支配规则

给定两个候选解 `a` 和 `b`，其约束违背度分别为 `cv_a` 和 `cv_b`，目标向量分别为 `F_a` 和 `F_b`。

本发明优选采用如下支配规则：

1. 如果 `cv_a < cv_b`，则 `a` 优于 `b`；
2. 如果 `cv_a = cv_b`，则比较多目标支配关系；
3. 若 `F_a` 各目标不差于 `F_b` 且至少一个目标更优，则 `a` 支配 `b`。

这意味着约束可行性优先于目标最优性。

该环境选择底座对应 [base.py](d:/VS%20Code/code/KEMM%28renew%29/kemm/algorithms/base.py#L26) 和 [base.py](d:/VS%20Code/code/KEMM%28renew%29/kemm/algorithms/base.py#L77)。

#### 5.8.2 拥挤距离

对于同一前沿中的解，优选采用拥挤距离维持解集多样性。

对每个目标维度 `m`，按目标值排序后，对内部点计算归一化间距增量：

`CD_i += (f_{i+1}^m - f_{i-1}^m) / (f_max^m - f_min^m + epsilon)`

边界点的拥挤距离设为无穷大，以保证极端解保留。

### 5.9 环境变化响应主流程

本发明的核心不只是普通进化，而是在环境变化时执行统一的响应流程。

4. 生成各来源候选解；
5. 构造统一候选池；
6. 评价候选池并执行环境选择；
7. 根据恢复质量更新自适应分配器。

### 5.10 环境变化诊断机制

#### 5.10.1 前沿特征构造

本发明优选将当前 Pareto 前沿压缩为固定维度特征向量。

一种优选的 10 维特征构造为：

`phi_t = [mean(f1), mean(f2), std(f1), std(f2), q25(f1), q25(f2), span(f1), span(f2), n_pf, rho_pf]`

其中：
- `mean` 为均值；
- `std` 为标准差；
- `q25` 为 25 分位数；
- `span` 为最大值与最小值差；
- `n_pf` 为当前前沿点数；
- `rho_pf` 为前沿点密度。

优点在于：
- 特征维度固定，便于时间建模；
- 能同时描述前沿位置、分散程度和规模。

该构造对应 [adaptive.py](d:/VS%20Code/code/KEMM%28renew%29/kemm/core/adaptive.py#L408)。

#### 5.10.2 变化幅度计算

设相邻两个时刻的前沿特征分别为 `phi_t` 和 `phi_{t-1}`，则变化幅度可优选表示为：

`Delta_t = (1 / d_phi) * sum_j |phi_t(j) - phi_{t-1}(j)| / (|phi_{t-1}(j)| + epsilon)`

其中：
- `d_phi` 为特征维度；
- `epsilon` 为避免分母为零的极小常数。

当需要平滑变化幅度时，可使用最近若干步的均值：

`M_t = clip(mean(Delta_{t-k+1}, ..., Delta_t), 0, 1)`

该变化幅度对应 [adaptive.py](d:/VS%20Code/code/KEMM%28renew%29/kemm/core/adaptive.py#L508)。

#### 5.10.3 可迁移性估计

本发明优选综合变化幅度和历史稳定性来估计知识可迁移性。

一种优选形式为：

`S_change = 1 - clip(3 * M_t, 0, 1)`

如果有历史 IGD 或等价质量代理值序列 `g_1, g_2, ..., g_t`，则可定义：

`CV_g = std(g) / (mean(g) + epsilon)`

`S_stable = 1 - clip(CV_g, 0, 1)`

则可迁移性可表示为：

`T_t = clip(eta_1 * S_change + eta_2 * S_stable, T_min, T_max)`

其中：
- `eta_1 + eta_2 = 1`；
- `T_min` 与 `T_max` 为可迁移性下上限。

在优选实现中：

`T_t = clip(0.6 * S_change + 0.4 * S_stable, 0.1, 0.9)`

对应 [adaptive.py](d:/VS%20Code/code/KEMM%28renew%29/kemm/core/adaptive.py#L520)。

其技术含义为：
- 变化越小，旧知识越可能可迁移；
- 历史表现越稳定，迁移可信度越高。

### 5.11 自适应分配机制

本发明优选采用带上下文的多臂老虎机策略，为四类恢复机制分配预算。

设四个策略臂分别为：

- `a_1 = memory`
- `a_2 = prediction`
- `a_3 = transfer`
- `a_4 = reinit`

#### 5.11.1 上下文划分

将变化幅度 `M_t` 映射到离散上下文桶：

`ctx_t = min(floor(M_t * N_ctx), N_ctx - 1)`

其中：
- `N_ctx` 为上下文数量。

这样做的含义是：
- 小变化和大变化不共享同一套策略历史；
- 不同变化区间能逐步学到不同的优选恢复模式。

#### 5.11.2 UCB 评分

在给定上下文下，对每个策略臂 `i`，定义：

`Q_i = mean(reward_window_i)`

`UCB_i = Q_i + c * sqrt( ln(T_total + 1) / (N_i + epsilon) )`

其中：
- `Q_i` 为最近窗口内平均奖励；
- `c` 为探索系数；
- `T_total` 为总更新次数；
- `N_i` 为第 `i` 个策略被更新次数。

该形式对应经典 UCB1，并已在实现中落地，见 [adaptive.py](d:/VS%20Code/code/KEMM%28renew%29/kemm/core/adaptive.py#L20)。

#### 5.11.3 比例化输出

为了把 `UCB_i` 转换为连续预算比例，本发明优选采用 softmax：

`p_i = exp((UCB_i - UCB_max) / tau) / sum_j exp((UCB_j - UCB_max) / tau)`

其中：
- `tau` 为温度参数。

再施加最小比例约束：

`p_i = max(p_i, p_min)`

并归一化：

`hat(p_i) = p_i / sum_j p_j`

最后得到：

`p = [p_memory, p_prediction, p_transfer, p_reinit]`

对应 [adaptive.py](d:/VS%20Code/code/KEMM%28renew%29/kemm/core/adaptive.py#L95)。

#### 5.11.4 离散预算分配

设当前规划种群规模为 `N`，则四类候选数量可写为：

`N_mem = floor(N * p_memory)`

`N_pred = floor(N * p_prediction)`

`N_trans = floor(N * p_transfer)`

`N_reinit = N - N_mem - N_pred - N_trans`

对应 [kemm.py](d:/VS%20Code/code/KEMM%28renew%29/kemm/algorithms/kemm.py#L323)。

#### 5.11.5 奖励更新

设当前恢复质量代理为 `Q_cur`，上一轮恢复质量代理为 `Q_prev`。

当质量值越大越好时，奖励可优选定义为：

`r_t = max(0, (Q_cur - Q_prev) / max(|Q_prev|, epsilon))`

若需裁剪到 `[0,1]`，则：

`r_t = clip(r_t, 0, 1)`

为了避免所有策略“平均领功”，本发明优选采用主导策略优先归因：

- 对占比最大的策略赋予完整奖励；
- 对其他高占比策略按比例赋予缩减奖励。

对应 [adaptive.py](d:/VS%20Code/code/KEMM%28renew%29/kemm/core/adaptive.py#L317)。

技术意义在于：
- 让系统学到“什么变化类型下该优先信任哪种恢复机制”；
- 避免固定比例的人工启发式。

### 5.12 历史记忆机制

本发明优选采用压缩式历史精英记忆机制，而不是保存全部历史种群。

#### 5.12.1 记忆库存储对象

记忆条目优选包括：
- 历史精英解的压缩潜空间统计量；
- 历史精英解对应的目标值统计；
- 环境指纹；
- 少量原始精英样本；
- 年龄信息。

其优选数据结构包括：

`M_j = { z_mean, z_std, fitness_mean, fitness_std, fingerprint, raw_elites, age }`

#### 5.12.2 环境指纹

环境指纹用于检索相似历史环境。

本发明优选通过种群统计和精英目标统计构造环境指纹，例如：

`fp_t = concat( mean(P_t)[:h], mean(F_elite_t), std(F_elite_t) )`

其中：
- `P_t` 为当前种群；
- `F_elite_t` 为当前精英解目标值；
- `h` 为从决策空间统计中截取的前部维度。

#### 5.12.3 VAE 压缩原理

本发明优选使用轻量变分自编码器压缩历史精英结构。

编码分布可写为：

`q_phi(z|x) = N(mu_phi(x), diag(sigma_phi^2(x)))`

解码分布可写为：

`p_theta(x|z)`

通过重参数化采样：

`z = mu + sigma ⊙ epsilon`

其中：
- `epsilon ~ N(0, I)`。

优选训练目标为 beta-VAE 形式：

`L_VAE = L_recon + beta * L_KL`

其中：

`L_recon = E ||x - x_hat||^2`

`L_KL = KL(q_phi(z|x) || N(0, I))`

展开为：

`L_KL = -1/2 * sum(1 + log sigma^2 - mu^2 - sigma^2)`

对应实现见 [memory.py](d:/VS%20Code/code/KEMM%28renew%29/kemm/core/memory.py#L519) 及其底层轻量 VAE。

#### 5.12.4 在线存储流程

对每个历史环境的精英解集 `E_t`，优选执行如下过程：

1. 用 `E_t` 对编码器做在线微调；
2. 将 `E_t` 编码到潜空间，得到 `z_mean, z_std`；
3. 保留少量 `raw_elites` 作为近似原型；
4. 按先进先出或容量控制写入记忆库。

若记忆库容量为 `C`，则：

当 `|M| < C` 时，直接写入；

当 `|M| >= C` 时，删除最旧条目再写入。

#### 5.12.5 相似环境检索

设查询指纹为 `fp_q`，记忆库中第 `j` 个条目的指纹为 `fp_j`。

为抵消不同特征量纲差异，先做尺度归一化：

`d_j = || (fp_j - fp_q) / (std(FP_all) + epsilon) ||_2`

相似度定义为：

`sim_j = 1 / (1 + d_j)`

选取 top-K 相似条目进入候选恢复。

对应 [memory.py](d:/VS%20Code/code/KEMM%28renew%29/kemm/core/memory.py#L635)。

#### 5.12.6 解码恢复

对每个命中的记忆条目，可从潜空间采样：

`z^(m) = z_mean + z_std ⊙ epsilon^(m)`

经解码器得到候选解：

`x^(m) = Decoder(z^(m))`

优选地，将解码样本与少量历史原始精英样本叠加，形成混合记忆候选池：

`C_mem = C_decode U C_raw`

其技术意义在于：
- 保留结构泛化能力；
- 同时维持少量真实精英原型，减小完全解码带来的漂移误差。

### 5.13 前沿预测机制

本发明优选对 Pareto 前沿的低维统计特征进行时间序列建模，而非直接预测单个最优点。

#### 5.13.1 特征序列

设每个历史时刻 `t` 的前沿特征向量为 `phi_t`，则得到特征序列：

`{phi_1, phi_2, ..., phi_t}`

#### 5.13.2 高斯过程回归

对每一维特征 `phi^(j)`，优选建立一个独立的高斯过程回归模型：

`phi^(j)(t) ~ GP(m_j(t), k_j(t, t'))`

优选核函数为径向基函数：

`k(t,t') = sigma_f^2 * exp( - (t - t')^2 / (2 l^2) )`

其中：
- `sigma_f^2` 为信号方差；
- `l` 为长度尺度。

设训练输入为 `X = [t_1, t_2, ..., t_n]^T`，训练输出为 `y_j`，则对下一个时刻 `t_*` 的预测均值和方差可写为：

`mu_* = k_*^T (K + sigma_n^2 I)^(-1) y_j`

`sigma_*^2 = k(t_*,t_*) - k_*^T (K + sigma_n^2 I)^(-1) k_*`

其中：
- `K` 为训练样本之间的核矩阵；
- `k_*` 为测试点与训练点之间的核向量；
- `sigma_n^2` 为噪声方差。

对应 [drift.py](d:/VS%20Code/code/KEMM%28renew%29/kemm/core/drift.py#L261)。

#### 5.13.3 预测置信度

本发明优选结合以下因素计算预测置信度：
- 历史样本长度；
- 特征随时间变化的规律性；
- 预测方差大小。

一种优选形式为：

`Conf_pred = 0.4 * S_hist + 0.6 * S_reg`

其中：
- `S_hist = min(n_hist / N_hist_max, 1)`；
- `S_reg` 为基于线性拟合优度或规律性评估得到的分值。

对应 [drift.py](d:/VS%20Code/code/KEMM%28renew%29/kemm/core/drift.py#L444)。

#### 5.13.4 预测候选生成

对下一个时刻预测得到前沿特征均值 `phi_hat_{t+1}` 后，优选围绕当前种群中心或精英中心生成预测候选。

设当前锚点集合为 `A`，其中心为：

`c_A = mean(A)`

扩散尺度为：

`s_A = max(std(A), s_floor)`

预测标准差平均值为 `sigma_bar`，则候选扩散尺度优选为：

`s_pred = s_A * (1 + 0.5 * clip(sigma_bar, 0.05, 2.5))`

于是预测候选可表示为：

`x_pred^(m) = c_A + n^(m) ⊙ s_pred`

其中 `n^(m)` 为标准高斯噪声。

当预测置信度不足时，本发明优选退化为低成本线性外推。

#### 5.13.5 线性外推

若历史质心序列为 `c_{t-2}, c_{t-1}, c_t`，则可优选构造：

速度项：

`v_t = c_t - c_{t-1}`

加速度项：

`a_t = 0.5 * (c_t - 2 c_{t-1} + c_{t-2})`

则预测中心可表示为：

`c_{t+1}^pred = clip(c_t + v_t + a_t, l, u)`

该线性外推对应 [kemm.py](d:/VS%20Code/code/KEMM%28renew%29/kemm/algorithms/kemm.py#L534)。

### 5.14 跨环境迁移机制

本发明优选通过子空间和流形映射实现历史环境向当前环境的结构迁移，而非直接复制旧解。

#### 5.14.1 多源环境选择

对检索得到的多个历史环境，根据相似度选择多个源环境：

`S = { S_1, S_2, ..., S_m }`

每个源环境 `S_j` 包括：
- 历史精英解集；
- 对应相似度 `sim_j`。

#### 5.14.2 相似度归一化权重

优选定义迁移权重：

`w_j = sim_j / sum_k sim_k`

其中 `w_j` 满足：

`sum_j w_j = 1`

这对应多源加权迁移，见 [transfer.py](d:/VS%20Code/code/KEMM%28renew%29/kemm/core/transfer.py#L523)。

#### 5.14.3 聚类与局部结构提取

对每个源环境的历史精英解集，优选先做聚类，将其分成 `L` 个局部簇：

`S_j = U_{q=1}^L S_{j,q}`

聚类的目的在于：
- 避免用单一全局线性结构拟合所有历史精英；
- 保留局部 Pareto 结构。

#### 5.14.4 PCA 子空间构建

对源簇 `S_{j,q}` 和当前环境锚点集合 `T` 分别做 PCA。

设源子空间和目标子空间基底分别为：

`P_S in R^(D x d)`

`P_T in R^(D x d)`

其中 `D` 为决策变量维度，`d` 为子空间维度。

子空间维度优选根据累计方差贡献率确定：

`d = min {k | sum_{i=1}^k lambda_i / sum_i lambda_i >= theta_var }`

其中：
- `lambda_i` 为 PCA 特征值；
- `theta_var` 为累计方差阈值，例如 0.95。

#### 5.14.5 Grassmann 流形测地流

本发明优选在 Grassmann 流形上构造源子空间与目标子空间之间的中间子空间序列。

先计算：

`M = P_S^T P_T`

对 `M` 做奇异值分解：

`M = U Sigma V^T`

主角向量定义为：

`Theta = arccos(Sigma)`

设 `R_S` 为 `P_S` 相对 `P_T` 的正交补，则测地流上的中间子空间可优选表示为：

`Phi(t) = P_S U cos(t Theta) + R_S V sin(t Theta)`

其中：
- `t in (0,1)`；
- 可以离散取若干个中间点 `t_k = k / (p+1)`。

该形式对应 [transfer.py](d:/VS%20Code/code/KEMM%28renew%29/kemm/core/transfer.py#L14)。

#### 5.14.6 投影与重建

对源簇中的样本 `x`，先做中心化：

`x_c = x - mu_S`

在中间子空间 `Phi(t_k)` 上投影：

`y_k = x_c^T Phi(t_k)`

再在目标环境附近重建：

`x_k^trans = y_k Phi(t_k)^T + mu_T`

其中：
- `mu_S` 为源簇均值；
- `mu_T` 为目标锚点均值。

这样就得到从源环境映射到目标环境附近的迁移候选。

#### 5.14.7 多源加权输出

对各源环境 `j` 生成的迁移候选集合 `C_j^trans`，按权重 `w_j` 分配输出数量：

`N_j = max(1, floor(N_trans * w_j))`

最终迁移候选池可表示为：

`C_trans = U_j Sample(C_j^trans, N_j)`

其技术意义在于：
- 当多个历史环境与当前环境都有一定相似性时，不必只依赖单一历史源；
- 提高迁移候选的多样性和稳健性。

### 5.15 统一候选池构造

本发明优选将多类候选解统一纳入候选池，而不让任何单个模块直接决定最终规划结果。

设：
- `C_mem` 为记忆候选；
- `C_pred` 为预测候选；
- `C_trans` 为迁移候选；
- `C_reinit` 为重初始化候选；
- `C_elite` 为精英保留候选；
- `C_prev` 为历史种群保留候选。

则统一候选池可表示为：

`C_all = C_mem U C_pred U C_trans U C_reinit U C_elite U C_prev`

在实际实现中，各集合可按矩阵拼接方式构造，并统一裁剪到变量边界区间。

对应 [kemm.py](d:/VS%20Code/code/KEMM%28renew%29/kemm/algorithms/kemm.py#L409) 之后的候选构造流程。

该统一候选池设计具有以下技术效果：
- 防止某单一恢复机制误判时直接污染最终解；
- 使不同知识来源在统一评价体系下公平竞争；
- 在多目标和约束条件下更稳定。

### 5.16 恢复质量估计

为了更新自适应分配器，本发明优选对当前变化响应质量进行估计。

在一种优选实现中，若当前 Pareto 前沿包含多个点，则对前沿点间最邻近距离取平均：

`Q_resp = mean_i min_{j != i} dist(F_i, F_j)`

若前沿规模较小，可退化为对当前种群某目标统计量的均值进行估计。

该质量代理用于比较相邻两轮变化响应的恢复效果，进而驱动自适应分配更新。

### 5.17 ship 场景中的代表轨迹选择

多目标优化得到的是轨迹解集，而实际执行需要一个代表轨迹。

本发明优选采用安全优先的代表轨迹选择策略。

#### 5.17.1 目标归一化分数

设轨迹解集中第 `i` 个候选轨迹的目标向量为 `F_i`。

对各目标维度做归一化：

`F_i^norm(m) = (F_i(m) - F_min(m)) / (F_max(m) - F_min(m) + epsilon)`

给定权重向量 `w`，则综合分数为：

`Score_i = sum_m w_m * F_i^norm(m)`

#### 5.17.2 安全优先排序

本发明优选在排序时先比较安全因素，再比较目标综合得分。

排序优先级可为：

1. 是否存在硬侵入；
2. 净距不足程度；
3. 是否到达目标；
4. 终端偏差；
5. 最小船间距离；
6. 归一化综合目标分数；
7. 最大风险。

该设计的技术意义在于：
- 保证最终执行轨迹首先满足动态航行安全要求；
- 再在安全范围内兼顾效率。

对应 [selection.py](d:/VS%20Code/code/KEMM%28renew%29/ship_simulation/optimizer/selection.py#L27)。

### 5.18 滚动重规划执行机制

本发明优选采用滚动重规划而非一次性全局规划。

#### 5.18.1 局部规划与局部执行

设局部规划时域为 `H_plan`，实际执行时域为 `H_exec`，且：

`0 < H_exec <= H_plan`

在第 `t` 个规划步：

1. 基于当前环境构造局部优化问题；
2. 运行本发明的多目标优化方法得到当前局部轨迹解集；
3. 从中选择代表轨迹；
4. 只执行代表轨迹的前 `H_exec` 部分；
5. 更新本船状态和目标船状态；
6. 进入下一次规划步。

#### 5.18.2 终止条件

在优选实现中，满足以下任一条件时可结束当前 episode：
- 本船到达目标区域；
- 达到最大重规划步数；
- 风险高于预设截止阈值；
- 或者任务完成条件成立。

该流程对应 [episode.py](d:/VS%20Code/code/KEMM%28renew%29/ship_simulation/optimizer/episode.py#L318)。

其技术意义在于：
- 适应动态环境；
- 避免一次性规划对长期预测误差过度敏感；
- 更符合实际自动航行控制逻辑。

为了避免将本发明保护范围不当地限定在某一种具体算法，本发明允许如下替代。

#### 5.24.1 记忆模块替代

除变分自编码器外，还可采用：
- 其他自编码器；
- 主成分压缩；
- 低秩表示；
- 聚类原型存储；
- 其他历史精英结构压缩方式。

#### 5.24.2 预测模块替代

除高斯过程回归外，还可采用：
- 自回归模型；
- 卡尔曼滤波；
- 贝叶斯回归；
- 递推最小二乘；
- 其他适于建模前沿特征时间演化的预测方式。

#### 5.24.3 迁移模块替代

除 Grassmann 测地流外，还可采用：
- 子空间对齐；
- 流形对齐；
- 特征空间投影；
- 域适应映射；
- 其他适于跨环境结构变换的迁移方式。

#### 5.24.4 自适应分配模块替代

除上下文多臂老虎机外，还可采用：
- 强化学习策略选择；
- 贝叶斯优化；
- 自适应规则表；
- 其他基于变化信息的动态预算分配方式。

#### 5.24.5 应用场景替代

本发明不限于船舶，还可扩展到：
- 无人艇；
- 无人车；
- 机器人动态规划；
- 其他动态多目标技术场景。

但本案优选的首次申请仍以船舶动态航行环境为主要保护对象。

## 六、可直接转摘要与发明内容的简版文本

为便于后续从说明书中抽取正式申请文本，给出一版可压缩使用的简版概述：

本发明公开了一种面向动态航行环境的船舶多目标轨迹优化方法、装置、电子设备及计算机程序产品。该方法获取本船、目标船、静态障碍物及环境场数据，构建以燃油、时间和风险为目标并带有约束违背度的动态轨迹优化模型；根据历史前沿特征确定环境变化信息，并据此自适应分配历史记忆候选、前沿预测候选、跨环境迁移候选和重初始化候选的生成比例；进一步构建统一候选池，执行约束支配环境选择，获得当前规划解集；从所述规划解集中确定代表轨迹并输出控制结果，用于滚动重规划。该方案能够提高环境变化后的轨迹恢复速度，提升动态航行场景中的安全性、稳定性和规划效率。
