## 1. 总体绘图原则

### 1.1 图面风格

建议采用专利常用的黑白线框风格或低饱和简化风格，不建议直接使用论文图或彩色 dashboard 图。

建议：

- 主流程采用矩形框 + 箭头
- 系统结构采用模块框图
- 场景图采用简化平面示意图
- 风险场、流场、候选池只表达结构，不追求论文级细节

不建议：

- 直接放 Matplotlib/Plotly 论文图
- 渐变色过多
- 复杂阴影和装饰元素
- 大段英文标注

### 1.2 图号规划

建议固定为以下 8 幅：

- 图1 本发明总体系统框图
- 图2 本发明方法流程图
- 图3 环境变化诊断与自适应分配示意图
- 图4 多源候选解协同生成示意图
- 图5 统一候选池与约束支配环境选择示意图
- 图6 滚动重规划执行示意图
- 图7 交叉会遇场景示意图
- 图8 港口高密障碍场景示意图

### 1.3 标注规则

- 模块标号使用 `100、110、120……`
- 物理对象标号使用 `210、220、230……`
- 时间窗口、规划窗口、执行窗口使用 `270、280……`
- 同一标号在所有图中保持同一含义

## 4. 各附图详细方案

### 4.1 图1 本发明总体系统框图

建议图面结构：

- 左侧为输入对象：
  - `210` 本船状态数据
  - `220` 目标船状态数据
  - `230` 静态障碍物数据
  - `240` 环境场数据
- 中间为处理模块链：
  - `100 -> 110 -> 120 -> 130`
  - `140、150、160、170` 以并联形式连接到 `180`
  - `180 -> 190 -> 200`
- 右侧为输出：
  - `250` 轨迹解集
  - `260` 代表轨迹
  - `290` 控制结果

建议箭头关系：

- `100` 输出到 `110`
- `110` 输出到 `120`
- `120` 输出到 `130`
- `130` 分别控制 `140/150/160/170`
- `140/150/160/170` 输出到 `180`
- `180` 输出到 `190`
- `190` 输出到 `200`

- [5.2 系统总体架构](d:/VS%20Code/code/KEMM%28renew%29/docs/patent_specification_ship_kemm_draft.md#L307)

### 4.2 图2 本发明方法流程图

建议流程步骤：

- `S100` 获取动态航行环境数据
- `S200` 构建动态多目标轨迹优化模型
- `S300` 归档当前环境精英信息
- `S400` 确定环境变化信息
- `S500` 确定候选数量分配比例
- `S600` 生成记忆候选
- `S700` 生成预测候选
- `S800` 生成迁移候选
- `S900` 构建统一候选池
- `S1000` 执行约束支配环境选择
- `S1100` 确定代表轨迹并执行滚动重规划

建议额外加一个循环回路：

- `S1100` 返回 `S100`

### 4.3 图3 环境变化诊断与自适应分配示意图

建议结构：

- 输入：
  - `310` 历史前沿特征序列
  - 当前前沿特征
- 中间：
  - 变化幅度计算
  - 可迁移性估计
  - 预测置信度估计
  - 上下文区间映射
  - `330` 策略评分器

  - `N_mem`
  - `N_pred`
  - `N_trans`

图面应表达的关键逻辑：

- 不是固定比例
- 而是“先诊断再分配”
