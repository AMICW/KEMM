# KEMM 船舶场景主案权利要求书初稿

本文档为内部初稿，用于和老师、代理人讨论保护范围。

建议主案名称：
- 一种面向动态航行环境的船舶多目标轨迹优化方法、装置、电子设备及计算机程序产品

说明：
- 本稿追求“先把保护骨架搭起来”，不是最终提交版。
- 独立权利要求尽量保持中等抽象层级，避免一上来写死 VAE、GP、Grassmann 等具体模型。
- 具体模型和替代实现建议保留在从属权利要求和说明书实施方式中。

---

## 权利要求书初稿

### 权利要求1

一种面向动态航行环境的船舶多目标轨迹优化方法，其特征在于，包括：

获取本船、目标船、静态障碍物及环境场数据；

根据所述本船、目标船、静态障碍物及环境场数据构建动态多目标轨迹优化模型，其中，所述动态多目标轨迹优化模型至少包括燃油目标、时间目标和风险目标，并包括用于表征安全约束违背程度的约束指标；

根据当前时刻与至少一个历史时刻对应的前沿特征，确定环境变化信息，所述环境变化信息至少包括变化幅度信息；

根据所述环境变化信息，确定多类候选解生成机制对应的分配比例，所述多类候选解生成机制至少包括历史记忆候选生成机制、前沿预测候选生成机制、跨环境迁移候选生成机制和重初始化候选生成机制；

基于所述分配比例，通过所述多类候选解生成机制分别生成候选解，并构建统一候选池；

对所述统一候选池执行约束支配环境选择，获得当前规划周期的轨迹解集；

根据所述轨迹解集确定代表轨迹，并输出所述代表轨迹对应的轨迹参数或控制结果，用于当前规划周期的船舶航行控制或滚动重规划。

### 权利要求2

根据权利要求1所述的方法，其特征在于，所述前沿特征包括以下至少之一：

各目标值的均值、各目标值的标准差、各目标值的跨度、前沿点数量、前沿密度、前沿位置特征、前沿分布特征。

### 权利要求3

根据权利要求1所述的方法，其特征在于，所述根据环境变化信息确定多类候选解生成机制对应的分配比例，包括：

将所述变化幅度信息映射到预设的上下文区间；

针对所述上下文区间对应的策略分配器，根据历史规划恢复效果确定所述历史记忆候选生成机制、前沿预测候选生成机制、跨环境迁移候选生成机制和重初始化候选生成机制的分配比例；

根据当前规划周期的恢复质量对所述策略分配器进行更新。

### 权利要求4

根据权利要求1所述的方法，其特征在于，所述基于历史记忆候选生成机制生成候选解，包括：

根据当前环境对应的环境指纹，从历史记忆库中检索与当前环境相似的历史环境条目；

从所述历史环境条目中提取历史精英解结构信息；

根据所述历史精英解结构信息生成记忆候选，并将所述记忆候选加入所述统一候选池；

其中，所述历史记忆库采用对历史精英解进行压缩存储的方式构建。

### 权利要求5

根据权利要求1所述的方法，其特征在于，所述基于前沿预测候选生成机制生成候选解，包括：

根据多个历史时刻的前沿特征序列建立前沿演化预测模型；

通过所述前沿演化预测模型预测下一规划时刻的前沿特征；

根据预测得到的前沿特征生成预测候选，并将所述预测候选加入所述统一候选池；

其中，当预测置信度低于预设阈值时，采用低复杂度外推方式或回退方式生成所述预测候选。

### 权利要求6

根据权利要求1所述的方法，其特征在于，所述基于跨环境迁移候选生成机制生成候选解，包括：

根据当前环境与多个历史环境之间的相似度确定一个或多个源环境；

提取所述源环境中的历史精英解；

根据所述源环境与当前环境之间的结构对应关系，对所述历史精英解执行子空间映射或流形映射，得到迁移候选；

按照所述相似度对多个源环境得到的迁移候选进行加权合并，并加入所述统一候选池。

### 权利要求7

根据权利要求1所述的方法，其特征在于，所述对统一候选池执行约束支配环境选择，包括：

根据各候选解对应的约束指标确定约束违背程度；

在约束违背程度不同的情况下，优先保留约束违背程度较低的候选解；

在约束违背程度相同或均满足约束的情况下，根据所述燃油目标、时间目标和风险目标之间的非支配关系确定保留候选解；

并基于解集分布信息保留多样性。

### 权利要求8

根据权利要求1所述的方法，其特征在于，所述根据轨迹解集确定代表轨迹，包括：

优先根据所述轨迹解集对应的安全净距或风险约束满足情况筛除不满足安全要求的候选轨迹；

在满足安全要求的候选轨迹中，根据燃油目标、时间目标和风险目标的综合排序确定所述代表轨迹；

输出所述代表轨迹对应的局部航路点、航段速度或控制参数。

### 权利要求9

一种面向动态航行环境的船舶多目标轨迹优化装置，其特征在于，包括：

数据获取模块，用于获取本船、目标船、静态障碍物及环境场数据；

问题建模模块，用于根据所述本船、目标船、静态障碍物及环境场数据构建动态多目标轨迹优化模型；

变化诊断模块，用于根据当前时刻与历史时刻对应的前沿特征确定环境变化信息；

分配控制模块，用于根据所述环境变化信息确定多类候选解生成机制对应的分配比例；

候选生成模块，用于基于所述分配比例生成历史记忆候选、前沿预测候选、跨环境迁移候选和重初始化候选，并构建统一候选池；

环境选择模块，用于对所述统一候选池执行约束支配环境选择，获得当前规划周期的轨迹解集；

结果输出模块，用于根据所述轨迹解集确定代表轨迹并输出所述代表轨迹对应的轨迹参数或控制结果；

其中，所述装置被配置为执行如权利要求1至8任一项所述的方法。

### 权利要求10

## 讨论意见

### 1. 当前权利要求1的定位

当前权利要求1的保护思路是：
- 不把发明写成纯算法；
- 明确限定在动态航行环境中的船舶轨迹优化；
- 抓住本案最核心的步骤链：
  数据获取 -> 问题建模 -> 变化诊断 -> 自适应分配 -> 多源候选生成 -> 统一环境选择 -> 轨迹输出。

这个链条基本能反映本案的本质。

### 2. 为什么没有在权1里直接写 VAE、GP、Grassmann

原因如下：
- 如果直接写死，保护范围会明显变窄。
- 审查阶段若现有技术中出现类似单模块实现，容易被卡。
- 对本案真正重要的是“多源知识增强 + 自适应分配 + 统一候选池选择”的整体流程，而不是某个模型名本身。

更合理的做法是：
- 权1写技术骨架；
- 从属写压缩记忆、预测模型、迁移模型等实现细节；
- 说明书再写足各种替代实现。

### 3. 后续可选的加密从属点

如果老师或代理人认为需要更强的技术特征，可以后续加以下从属点：
- 环境变化信息还包括可迁移性信息和预测置信度信息；
- 历史记忆库中的环境指纹由种群质心、前沿统计和风险统计构成；
- 预测候选由前沿特征预测结果与精英中心扩散生成；
- 迁移候选通过多个源环境的相似度加权组合得到；
- 约束指标由安全净距不足、硬侵入、越界和终端偏差共同构成；
- 所述方法应用于滚动重规划场景，仅执行代表轨迹的前段并在下一时刻重新规划。

### 4. 可能的第二套权1备选写法

如果后续希望更突出“恢复机制”，也可以将权1修改为：
- 将“统一候选池”和“恢复规划”写得更重；
- 将“环境变化后恢复轨迹搜索速度”写成更明确的技术目标；
- 将“滚动重规划”写成独立必要特征。

这套写法会更贴 ship，但保护范围会略窄。
